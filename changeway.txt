问题 1：点击 Download/Upload 不显示红色进度条
原因分析：

UI 层（ui_manager.c）中，on_download_btn_clicked 和 on_upload_btn_clicked 调用了 ui_show_progress_batch()。但在该函数中，给进度条设置红色指示器（LV_PART_INDICATOR）的代码存在逻辑覆盖或作用域问题，导致颜色未生效。
网络层（network_task.c）在启动单文件下载/上传时，调用的是 ui_show_progress()，但部分状态判断可能导致弹窗未正确弹出或被立即隐藏。
修改步骤：

打开 ui_manager.c，定位到 ui_show_progress_batch(void) 函数。
确保进度条创建后，明确设置指示器颜色为红色：
找到 lv_obj_set_style_bg_color(prog_slots[0].bar, lv_color_hex(0xCC0000), LV_PART_INDICATOR); 这一行。
确保没有后续代码将 LV_PART_INDICATOR 的背景色覆盖为其他颜色（如灰色或蓝色）。
打开 network_task.c，定位到 start_download 和 start_upload 函数。
在这两个函数中，找到类似 if (!g_batch_active) 的条件判断，确保无论是单文件还是批量文件，只要开始传输，都会调用 lv_async_call(cb_show_progress, spd); 来触发 UI 层的进度条显示。
确保 cb_show_progress 回调中调用的 ui_show_progress() 正确创建了带有红色指示器的 lv_bar。
问题 2：点击 Upload 后，文件未上传到服务器端
原因分析：

本地路径拼接错误：在 ui_manager.c 的 on_upload_btn_clicked 中，文件路径拼接逻辑存在缺陷。如果 g_local_cur_path 为空，它直接使用文件名，但在 network_task.c 的 network_cmd_put 中，硬编码了 ./client/%s。当 ui_manager.c 传入了带有 ./client/ 前缀的路径时，network_task.c 会拼接成 ./client/./client/xxx，导致找不到文件。
批量上传逻辑冲突：network_task.c 中的 network_cmd_put_multi 仅将任务推入队列，但 network_cmd_put（实际执行单文件发送）没有正确被网络主线程的循环调度。
修改步骤：

打开 ui_manager.c，定位到 on_upload_btn_clicked 函数。
修改文件名收集逻辑：不要在 UI 层加 ./client/ 前缀。直接把用户选中的文件名（不含路径）赋给 files[i]，或者只拼接相对子路径（如 subdir/file.txt），将完整路径的拼接工作全部交给网络层。
打开 network_task.c，定位到 network_cmd_put 函数。
修改文件存在性检查和路径拼接逻辑：
无论传入的 filename 是否带子目录，都统一拼接为 ./client/%s。
如果 filename 已经包含 ./client/，则不要重复拼接。
打开 network_task.c，定位到 network_thread_func 的主循环。
检查 if (g_state == ST_IDLE && g_tx_queue.count > 0) 逻辑块，确保调用了 batch_start_next()。
在 batch_start_next() 中，如果是上传任务，确保调用的是 network_cmd_put(task.filename)，并且 network_cmd_put 能正确打开文件并进入 ST_WAIT_PUT_RESP 状态。
